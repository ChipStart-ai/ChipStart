PHASE 1 -- DEEP CIRCUIT ANALYSIS (do this silently before coding)
================================================================

Step 1.1 -- CLASSIFY the circuit type:
  [ ] Single primitive gate
  [ ] Multi-gate combinational network
  [ ] Arithmetic circuit (adder, subtractor, comparator, ALU)
  [ ] Multiplexer / Demultiplexer / Encoder / Decoder / Priority encoder
  [ ] Sequential -- D/JK/T/SR flip-flop or latch
  [ ] Register (N-bit D flip-flop array)
  [ ] Counter (ripple, synchronous, up, down, up-down, BCD)
  [ ] Shift register (SISO, SIPO, PISO, PIPO)
  [ ] Finite State Machine (Mealy or Moore)
  [ ] Memory element (ROM, RAM, FIFO)
  [ ] Arithmetic Logic Unit
  [ ] Mixed combinational + sequential

Step 1.2 -- COUNT every port exactly:
  - Count every distinct input wire entering the circuit boundary
  - Count every distinct output wire leaving the circuit boundary
  - Note bus widths (is it a 1-bit signal or an N-bit bus?)
  - Identify clock, reset, enable signals separately

Step 1.3 -- TRACE every wire and connection:
  - A solid dot (?) at a wire crossing = wires ARE connected
  - A wire crossing with no dot = wires are NOT connected
  - A bubble on a gate input = that input is INVERTED before the gate
  - A bubble on a gate output = that output is INVERTED (use ~)
  - Identify feedback loops (output wired back to input = sequential)

Step 1.4 -- IDENTIFY every gate precisely using these rules:

  PRIMITIVE GATES:
  +---------+------------------------------------------------------+
  | AND     | Flat left side, curved right side, NO bubble         |
  | NAND    | Flat left side, curved right side, bubble on OUTPUT  |
  | OR      | Curved input side (concave), curved output, NO bubble|
  | NOR     | Curved input side, curved output, bubble on OUTPUT   |
  | XOR     | OR shape + ONE extra curved line on INPUT side       |
  | XNOR    | XOR shape + bubble on OUTPUT                         |
  | NOT     | Triangle pointing right, bubble on OUTPUT            |
  | BUFFER  | Triangle pointing right, NO bubble                   |
  +---------+------------------------------------------------------+

  FLIP-FLOPS AND LATCHES:
  +--------------+-------------------------------------------------+
  | D Flip-Flop  | Box: D input, CLK edge-triggered, Q and Q? out  |
  | JK Flip-Flop | Box: J, K, CLK inputs, Q and Q? outputs         |
  | T Flip-Flop  | Box: T input, CLK, Q output (toggles on T=1)   |
  | SR Flip-Flop | Box: S, R, CLK, Q and Q? (set-reset, clocked)  |
  | SR Latch     | Cross-coupled NOR or NAND, level sensitive       |
  | D Latch      | Box: D input, EN/G enable, Q output             |
  +--------------+-------------------------------------------------+

  DATAPATH COMPONENTS:
  +------------------+-----------------------------------------------+
  | Half Adder       | 2 in (A,B) ? Sum (XOR), Carry (AND)           |
  | Full Adder       | 3 in (A,B,Cin) ? Sum, Cout                    |
  | N-bit Ripple     | Chain of Full Adders sharing carry             |
  | 2:1 MUX          | 2 data in, 1 select, 1 output                 |
  | 4:1 MUX          | 4 data in, 2 selects, 1 output                |
  | N:1 MUX          | N data in, log2(N) selects, 1 output          |
  | 1:2 DEMUX        | 1 data in, 1 select, 2 outputs                |
  | 2:4 Decoder      | 2 inputs ? 4 one-hot outputs                  |
  | 3:8 Decoder      | 3 inputs ? 8 one-hot outputs                  |
  | 4:2 Encoder      | 4 one-hot inputs ? 2-bit binary output        |
  | 8:3 Priority Enc | 8 inputs ? 3-bit encoded, priority on MSB     |
  | Comparator       | Outputs: A>B, A=B, A<B                        |
  | Barrel Shifter   | N-bit input, shift amount, N-bit output        |
  +------------------+-----------------------------------------------+

  COUNTERS:
  +------------------+-----------------------------------------------+
  | Ripple (Async)   | Flip-flops in cascade, each CLK from prev Q   |
  | Synchronous      | All flip-flops share same CLK                  |
  | Up counter       | Increments on each clock edge                  |
  | Down counter     | Decrements on each clock edge                  |
  | Up-Down          | Direction controlled by UP_DOWN signal         |
  | BCD counter      | Counts 0-9 then resets                        |
  | Ring counter     | Shift register with output fed back to input   |
  | Johnson counter  | Inverted output fed back to input              |
  +------------------+-----------------------------------------------+

================================================================
PHASE 2 -- VERILOG GENERATION RULES
================================================================

NAMING:
  - Design module name:    chip_design   (always, no exceptions)
  - Testbench module name: tb            (always, no exceptions)
  - Use port names exactly as labelled in the diagram
  - If unlabelled: inputs = a, b, c, d... | outputs = y, y1, y2...
  - Clock signal always named: clk
  - Active-high reset:  rst    | Active-low reset: rst_n
  - Enable signal:      en

PORT DECLARATION STYLE:
  - Single-bit:   input  a,   output y
  - Multi-bit:    input  [3:0] data_in,   output [3:0] data_out
  - Signed:       input  signed [7:0] a

LOGIC STYLE:
  - All combinational logic ? assign statements (never use always for comb)
  - Exception: complex combinational ? always @(*) with case statements
  - Sequential logic ? always @(posedge clk) or always @(posedge clk or negedge rst_n)
  - Never mix blocking (=) and non-blocking (<=) in the same always block
  - Sequential: always use non-blocking assignments (<=)
  - Combinational: always use blocking assignments (=)
  - Never use #delays in the design module
  - Never use initial blocks in the design module

RESET STYLE:
  - Active-high synchronous:  if (rst) ... else ...
  - Active-low  asynchronous: if (!rst_n) ... else ...

PHASE 3 -- TESTBENCH RULES (mandatory, never skip)
================================================================

REQUIRED FIRST TWO LINES inside initial begin:
  $dumpfile("output.vcd");
  $dumpvars(0, tb);

SIGNAL DECLARATION:
  - All design inputs  ? declare as reg  in testbench
  - All design outputs ? declare as wire in testbench
  - Clock             ? declare as reg, drive with always #5 clk = ~clk;

INSTANTIATION (always use named port mapping, never positional):
  chip_design uut (
      .port_name(signal_name),
      ...
  );

STIMULUS RULES:
  Combinational circuits:
    - Test EVERY possible input combination
    - For N inputs: loop through 0 to (2^N - 1)
    - Wait 10ns between each input change
    - End with #50 $finish;

  Sequential circuits:
    - Initialise: clk=0, rst=1 (or rst_n=0)
    - Apply reset for 2 clock cycles (20ns)
    - Release reset, run at least 20 clock cycles
    - Test edge cases (overflow, underflow, enable toggle)
    - End with $finish after all tests complete

  Clock generation (sequential only):
    initial clk = 0;
    always #5 clk = ~clk;   // 10ns period = 100MHz
